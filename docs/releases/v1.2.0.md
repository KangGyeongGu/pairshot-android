## Added

- 인터랙티브 튜토리얼 시스템 — 메인 16 단계 + 내보내기 설정 4 단계 contextual 가이드, 설정에서 "튜토리얼 다시 보기" 진입점 (`tutorial`)
- 카메라 비율 (4:3 / 16:9 / 1:1) 토글 UI, Before/After 비율 잠금, 광고 배너 오버레이 (`camera`)
- CameraX ResolutionSelector + 1:1 중앙 크롭 적용 (`camera`)
- 카메라 비율 enum 및 페어 영속 필드 추가 (`model`)
- Google Play 구독 + 엔타이틀먼트 통합 시스템 (`iap`)
- IAP 후속 통합 정비 — 기능별 Pro 게이팅, 결제 에러/고지, UI 리디자인 (`iap`)
- UMP 광고 동의 SDK 통합 — 앱 시작 동의 폼 + 설정 동의 변경 진입점 (`ads`)
- 쿠폰 → 프로모션 시스템 마이그레이션, Membership 단일 모델 (`promotion`)
- 개별 사진에 워터마크 외 테두리·라벨 데코레이션 적용 (`export`)
- 디자인 토큰 체계 정착 — primitive · component · 화면 invariant 3 계층 (`designsystem`)
- 무료 한도·잠금 기능 차단 시 paywall 에 컨텍스트 스낵바 안내 (`pro-guidance`)
- 내보내기 화질 프리셋 · 페어 병렬 처리 + 미리보기 일관성 (`export`)

## Changed

- 내보내기 기본 화질 프리셋 `BEST` (원본 해상도, JPEG 100%) 로 변경 — 신규 설치 한정 (`settings`)
- 설정 기본값을 데이터 클래스 단일 source 로 응집, enum 11 개에 `DEFAULT` + `fromName` 표준화 (`settings`)
- 흐름 제어 중앙화 — `AppFlowCoordinator` + `AppFlowDecider` 도입, 설정/튜토리얼/페이월 분기 단일 평가 (`flow`)
- `AppTheme` 을 `:core:model` 로 이동, DataStore 가 enum DEFAULT 참조 (`settings`)
- 카메라 홈/썸네일 진입 버튼을 명확한 홈 아이콘으로 변경 (`camera`)

## Fixed

- 워터마크 로고 등록 후 변경/제거 불가 문제 해소 — 단일 행에 파일명 + 변경/제거 아이콘 (`settings`)
- 카메라 화면 immersive 모드에서 스낵바가 화면 최상단에 붙던 정렬 문제 (`camera`)
- AFTER 촬영 완료 시 자동 navigate 제거, 스낵바 emit 지연 해소 (`camera`)
- 일일 한도 초과 free 유저가 튜토리얼 진입 못하던 문제 (`tutorial`)
- 튜토리얼 종료 시 sandbox 페어가 잠시 노출되던 시각적 race, 프로세스 death 시 ghost 페어 누수 (`tutorial`)
- AFTER 촬영 dormant 단계에서 홈/설정 버튼이 다시 활성되던 문제 (`tutorial`)
- `PhotoPairRepositoryImpl.delete` · `MediaStoreManager.deleteFromGallery` 의 예외 처리 견고화 (`data`)
- destructive action 시각 강도 보강 — error 색을 또렷한 빨강으로 (`designsystem`)
- release signingConfigs 가 키스토어 미보유 환경에서 NPE 던지지 않도록 가드 (`build`)
- 정밀 검수 후속 정리 — 만료일 추정 제거, Camera2 격리, 스레드 I/O 수정 (`refactor`)

## Compatibility

- **Android**: API 26+ (Android 8.0+)
- **Device**: Phone (Portrait only)
- **Data migration**: Room v2 → v3 자동 (`photo_pairs.aspectRatio TEXT` 컬럼 추가) / MigrationTest 5건 통과 (실기기 검증). DataStore 신규 키는 모두 default fallback. 레거시 `jpeg_quality` → `image_quality` 자동 매핑 (≤80 → LOW, ≥93 → BEST, else HIGH).
